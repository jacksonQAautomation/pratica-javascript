# 🧪 Laboratório de JavaScript para QA

Bem-vindo ao laboratório `pratica-javascript`!
Este projeto é meu ambiente de estudos do **Módulo 2 do curso de Automação com Playwright**.
O objetivo principal aqui é praticar o “volante” da automação: entender o fluxo, a lógica e os pilares do JavaScript para criar testes robustos.

---

## 🎯 Descrição

Este repositório não é para ser um projeto de “desenvolvedor full-stack”.
É um espaço de aprendizagem direcionado para QA, onde cada exercício e cada linha de código são usados para construir confiança em processos de automação e raciocínio técnico.

---

## 📚 Tópicos de estudo

- `Variáveis` (let, const)
- `Tipos de Dados` (string, number, boolean, object, array, null, undefined)
- `Arrays` (criação, manipulação e métodos)
- `Funções` (declaração, expressão, arrow, parâmetros e retorno)
- `Condicionais` (`if`, `else`, `switch`, operadores lógicos)
- `Loops` (`for`, `while`, `for...of`, `forEach`)
- `Async/Await` (fluxo assíncrono em testes, tratamento de promessas)

---

## 🧠 Mentalidade

> **“Profissional não confia. Profissional confere.”**

Essa é a base da automação para QA: nunca dar algo como certo sem validação, sempre verificar o comportamento e tornar cada fluxo repetível e confiável.

---

## 🚀 Objetivos do laboratório

- Aprender a escrever scripts legíveis e manuteníveis
- Compreender o ciclo de escrita/verificação/validação
- Dominar as ferramentas de controle de fluxo e execução assíncrona
- Construir base sólida para Playwright e automação de ponta a ponta

---

## 🛠️ Como usar

1. Clonar o repositório
2. Executar os exemplos em `node`
3. Evoluir para scripts no Playwright
4. Documentar aprendizados e dúvidas no próprio repositório

---

## 💡 Dica de progresso

- Faça commits pequenos e frequentes com mensagens claras
- Compartilhe o que aprendeu: perguntas e descobertas ajudam a fixar o conteúdo
- Refatore o código após a primeira versão para melhorar clareza e confiabilidade
